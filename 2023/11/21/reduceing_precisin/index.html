Reducing Precision

Introduction

优点：

  • reduced data movement (i.e., reduce memory bandwidth)
  • reduced storage cost (i.e., reduce chip area)
  • reduced energy per memory access (due to smaller memories)
  • reduced energy and time per MAC operation

缺点：

  • accuracy

复杂度：

  • 乘法器面积与功耗O(\(n^2\))
  • 乘法器延迟O(n)
  • 累加器面积与功耗O(n)
  • 累加器延迟O(n)或O(\(\sqrt{n}\))

Reducing the bit width of the fully accumulated partial sum will not have a significant impact on accuracy if the distribution of the weights and activations are centered near zero with a limited standard deviation; batch normalization may help achieve this effect.

define the bit-widths

  1. Quantization

    设量化台阶为 L，量化后的 value 为 \(q_i, 0\leq i \leq L-1\) 原数据为 \(x, x_{min}\leq x \leq x_{max}\) 则量化后的输出为 \(\hat{x}=Q(x)=q_i, d_i \lt x \leq d_{i+1}, d_{i+1}-d_i=d_i-d_{i-1}\) 量化误差为 \(E[(x-\hat{x})^2]=\int_{x_{min}}^{x_{max}}(x-\hat{x})^2\cdot p(x)dx\)

  2. 均匀量化

    \(q_i=ki+b\) 只有原数据满足均匀分布时，误差才最小

  3. 非均匀量化

    比如，log 量化 \(q_i=2^i\) 这样好处是 i 到 \(q_i\) 的映射在硬件上为移位，multiplier-free 同时，量化后的 value 相乘也为简单的移位 而其余的非线性量化，则将 i 映射到 \(q_i\) 时，需使用查找表(look-up table)或哈希表(hash table)，并且完成计算后转换为均匀区域的值

  4. 量化步骤

    1. 决定输入 x 的量化 index 值 i
    2. 将 i 映射为 \(q_i\)
    3. 对映射后的 \(q_i\) 完成 MAC 等计算

    量化方法本质是一个 精确度 与 硬件代价 的 trade-off

standard component of the bit width

一旦量化的值确定，我们就要使用一定的位数以及规则来表示

  1. 规则
    1. exponent: \(n_e\)-bits 用于表示的数的范围大小为 \(2^{e-127}\)，其中，\(e\)由\(n_e\)-bits的数表示，可正可负
    2. mantissa: \(n_m\)-bits 用于表示 \(2^{n_m}\) values
    3. sign: \(n_s\)=1位
    4. 因此，总位数：\(n_s+n_e+n_m\)
  2. 几种 StandardNumerical Representations
    1. Fixed point format(integer): fixed range，也就是不需要 \(n_e\)-bits \((-1)^s\times m\)
    2. Floating point(FP) format: 对于每个 value，range可以改变，也就是包含了 \(n_e\)bits \((-1)^s\times m \times 2^{(e-127)}\)
    3. 一般在CPU和GPU里用的是 fp32，而推断时，尝使用8-bit fixed point 或 16-bit float point进行乘法计算
  3. 硬件上实现乘法
    1. 定点数乘法 当然，改进的一般使用booth乘法器
    2. 浮点数乘法 在定点数乘法上，加入exponent的计算
    3. 目前有许多改进方法，通过 exponent sharing，比如 block floating point，dynamic fixed point等等，可看作是介于 floating point 和 fixed point 之间

custom numerical representations

实际上，是 manitissa(number of unique values) 与 exponent(range) 之间的 trade-off 比如，fp16中，使用\(n_s=1, n_e=5, n_m=10\)，而在 bfloat16(brain floating-point format)中，\(n_s=1, n_e=8, n_m=7\) 支持更大的范围有利于训练中的梯度求取 又比如，ms-fp8， ms-fp9 等等

mixed precision: different precision for different data types

对于 inference 来说，数据种类分为 weights, activations, partial sums 对于 training 来说，数据种类分为 weights, activations, partial sums, gradients, and weight updates Training typically requires higher precision than inference since the gradients and weights updates have a larger range then the weights andactivations

这时，硬件 MAC 需要支持位数的变化，因此需要 data-gated MAC

然而，大量计算单元将闲置，造成硬件资源浪费和吞吐量的减少。将 precision-scable MAC 类比 DNN 的 PE array
